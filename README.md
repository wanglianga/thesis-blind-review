# 高校论文盲审送审与意见回收平台

## 原始需求

> 请实现高校论文盲审送审与意见回收平台，React 管理端给研究生院、学院秘书、导师、学生和外审专家使用，Spring Boot 保存论文版本、查重结果、学科方向、回避名单、送审批次、专家邀请、评阅意见和答辩资格。学生提交匿名论文和材料；学院秘书初审格式、方向和回避人员；研究生院匹配专家、监控回收进度和处理超期；导师确认修改说明；学生根据意见上传新版。这个产品要守住匿名和回避规则，同时让学院知道论文卡在格式、专家、意见还是学生修改，不要让答辩资格靠人工表格传来传去。

## 项目简介

本平台是一个面向高校的全流程论文盲审管理系统，支持研究生院、学院秘书、导师、学生和外审专家五种角色协同工作，实现论文从提交、初审、专家匹配、外审评阅、修改到答辩资格认定的全流程数字化管理。

### 核心特性

- **双盲评审机制**：严格的匿名化处理，确保评审公正性
- **回避名单管理**：支持学生申请回避专家，学院审核确认
- **全流程可视化**：清晰的流程追踪，实时掌握论文当前阶段
- **多角色协同**：研究生院、学院秘书、导师、学生、外审专家各司其职
- **答辩资格自动认定**：基于评阅结果自动判断答辩资格，无需人工表格
- **专家超期补派**：专家拒审、超期或意见不合规时可补派新专家，保留原记录
- **重大修改复审**：重大修改论文进入复审流程，研究生院对比审核决定是否放行答辩

### 新功能说明

#### 专家超期补派
- 触发条件：专家拒审、超期未返回评审意见、意见格式不合规
- 补派操作：研究生院在评审进度页面点击"补派"按钮，选择新专家和送审批次
- 记录保留：原邀请记录的催办次数、拒审原因、补派原因等全部保留
- 学院秘书视角：论文管理页面显示最新的评审截止日期和超期提醒
- 学生端保护：盲审处理中阶段仅显示"盲审仍在处理中"，不暴露专家信息

#### 重大修改复审
- 自动识别：评阅意见中存在主要问题时自动标记为重大修改
- 学生提交：上传修改说明、差异说明和新版论文正文
- 导师确认：导师审核修改版本，确认后进入复审流程
- 多轮区分：第一轮意见和复审意见通过 `reviewRound` 字段区分，互不覆盖
- 研究生院审核：复审完成后进入研究生院人工审核，可对比第一轮意见、复审意见和学生修改回应，决定是否放行答辩

### 技术栈

- **后端**：Spring Boot 3 + Spring Security + MyBatis-Plus + MySQL 8
- **前端**：React 18 + Ant Design 5 + React Router 6 + Axios
- **部署**：Docker + Docker Compose

## 启动方式

### 前置要求

- Docker & Docker Compose
- 或者 Node.js 18+、JDK 17+、MySQL 8.0+

### Docker 一键启动（推荐）

#### 1. 启动服务

在项目根目录执行：

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问地址

- 前端管理端：http://localhost:3000
- 后端 API：http://localhost:8080/api

#### 3. 停止服务

```bash
docker compose down
```

如需清除数据：

```bash
docker compose down -v
```

### 本地开发启动

#### 1. 数据库准备

安装 MySQL 8.0+，创建数据库：

```sql
CREATE DATABASE thesis_review CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2. 启动后端

```bash
cd backend

# 编译打包
mvn clean package -DskipTests

# 运行
java -jar target/thesis-blind-review-1.0.0.jar
```

后端默认端口：8080，API 前缀：/api

#### 3. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端默认端口：3000，已配置代理转发 /api 到后端

## 核心业务流程

1. **学生提交**：学生提交论文（原文 + 匿名版），填写学科方向和回避名单
2. **学院初审**：学院秘书审核格式、方向、回避人员，通过或驳回
3. **专家匹配**：研究生院根据学科方向匹配专家，自动过滤回避名单
4. **专家评阅**：外审专家接受邀请，阅读匿名论文，提交评阅意见
5. **意见回收**：系统自动汇总评阅结果，判定答辩资格
6. **修改再送审**：如需修改，学生修改后由导师确认，重新送审
7. **资格认定**：全部通过则获得答辩资格，未通过则不具备答辩资格

## 注意事项

- 系统首次启动会自动初始化测试数据
- 论文文件上传为模拟实现，实际部署需接入对象存储
- 查重结果模块预留接口，可对接第三方查重系统
- 评审意见提交后专家不可修改，确保评审严肃性
